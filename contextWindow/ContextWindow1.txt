## 3) Frontend → Backend API Mapping
API base in frontend/src/app/data/api.ts:
- API base: VITE_API_BASE_URL || http://localhost:8000

Endpoints used:
- POST /analyze-video → createJob(file)
- GET /job-status/{job_id} → getJobStatus
- GET /job-result/{job_id} → getJobResult
- GET /events → getEvents (still exists but frontend now prefers job-result)
- GET /possession → getPossession (still exists but frontend now prefers job-result)
- GET /possession-timeline → getPossessionTimeline (still exists but frontend now prefers job-result)

Adapters (frontend/src/app/data/adapters.ts):
- adaptEvents: ApiEvent[] → UIEvent[]
- adaptPossessionSummary: {team: %} → array for charts
- adaptPossessionSegments: clock strings → seconds
- buildPlayerAnalytics: frames → player heatmap list

## 4) Frontend Interactivity (Job-Result Driven)
PlayerScoutingTool:
- Upload → POST /analyze-video → job_id
- Poll GET /job-status/{job_id}
- On completion: GET /job-result/{job_id}
- Uses job-result for:
  - events overlays
  - event timeline
  - possession summary + timeline
  - player heatmap
- Stores lastJobId in localStorage

TacticalAnalysisTool:
- Match selector (default: latest from localStorage)
- Fetches job-result with match_id (query param)
- Uses job-result for events, possession, segments, and players

## 6) Updated Backend Pipeline (Dynamic Job Results)
File: backend-fastapi/app/pipelines/video_pipeline.py
- Loads cv-models dynamically (cv_models)
- Runs VideoProcessor on uploaded video
- Runs EventEngine on frames
- Attaches player speed/distance from TacticalAnalyzer
- Enriches events with x/y from ball position
- Returns:
  - match_metadata
  - events
  - possession_summary
  - possession_segments
  - frames
